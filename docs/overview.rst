Обзор проекта
=============

Цель
----

Система автоматизации бронирования гостиниц. Позволяет:

* Туристам — выбирать отель, номер, оплачивать и получать подтверждение
* Менеджерам — управлять расписанием сотрудников и бронями
* Горничным — получать задания на уборку и отмечать выполнение

Аналоги на рынке
----------------

.. list-table::
   :header-rows: 1

   * - Система
     - СУБД
     - Архитектура
   * - Booking.com
     - PostgreSQL / Cassandra
     - Микросервисы
   * - Opera PMS (Oracle)
     - Oracle DB
     - Монолит / клиент-сервер
   * - Airtable-based PMS
     - Airtable (NoSQL)
     - SaaS / low-code

Данный проект использует **PostgreSQL** в монолитной архитектуре FastAPI — оптимально для учебного проекта и малой гостиничной сети.

Целевая аудитория
-----------------

* **Турист** — бронирование, просмотр рейтинга, оплата
* **Менеджер** — управление персоналом, отчёты, дашборд
* **Горничная (Cleaner)** — получение заданий, обновление статуса номеров
* **Администратор** — управление системой

Структура БД
------------

11 таблиц: ``hotels``, ``room_categories``, ``rooms``, ``guests``,
``guest_preferences``, ``guest_pref_link``, ``staff``, ``bookings``,
``services``, ``service_orders``, ``audit_log``.

Схема данных — см. :doc:`diagrams`.
